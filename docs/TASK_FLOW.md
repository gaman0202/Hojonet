# 타스크 처리 흐름 (유저 조작 순서 / 시나리오)

> 목적: 비개발자도 이해할 수 있도록 “전문가가 타스크를 만들고 → 각자 처리하고 → 전문가가 승인/기각까지 하는 흐름”을 단계별로 정리

---

## 0. 등장인물/용어

- **전문가**: 케이스(案件)를 진행하는 담당자(행정서사(expert)/어시스턴트(asistant) 포함)
- **유저**: 신청자(고객(customer),멤버(member))
- **타스크(Task)**: “누가 무엇을 언제까지 해야 하는지”를 나타내는 할 일

---

## 1) 타스크 생성 시나리오 (전문가가 만든다)

### Step 1. 전문가가 케이스 상세로 이동
1. 전문가가 케이스(案件) 상세 화면을 연다.
2. **[새 타스크 만들기]** 버튼을 누른다.

### Step 2. 전문가가 타스크 3개를 생성 할경우의 예
전문가는 아래 3개의 타스크를 만든다.

- **전문가 타스크 1 (전문가 담당)**
  - 예: “서류 초안 작성”
  - 기한/우선순위 설정 가능

- **유저 타스크 2 (유저 담당)**
  - 예: “재무제표 PDF 업로드”
  - 타입 예시: 파일 업로드 / 폼 작성
  - 기한/우선순위 설정 가능

- **전문가 타스크 3 (전문가 담당)**
  - 예: “유저 제출 자료 검토 후 승인/기각”
  - (선택) 타스크2와 “검토/승인” 관계로 연결될 수 있음

✅ **생성 직후 유저 화면에는 ‘유저 타스크 2’가 즉시 보이도록 한다.**  
(유저는 자기 담당 타스크만 확인/처리)

---

## 2) 처리 시나리오 (각자 자기 타스크를 처리한다)

### Step 3. 전문가가 타스크1을 처리
1. 전문가가 **전문가 타스크 1**을 열고 작업한다.
2. 작업 완료 후 **[완료 처리]** 버튼을 누른다.
3. 결과: 타스크1 상태가 **완료**로 변경된다.

### Step 4. 유저가 타스크2를 처리(제출)
1. 유저가 케이스 상세 화면으로 들어간다.
2. **유저 타스크 2**가 “요청됨/진행중”으로 표시된다.
3. 유저가 타스크2를 수행한다.
   - 파일 업로드 타입: 파일 업로드 → 제출
   - 폼 작성 타입: 폼 입력 → 제출
4. 유저가 제출하면 타스크2 상태는 **제출됨(검토 대기)** 로 바뀐다.
5. 전문가 화면에는 “검토 필요” 표시/알림이 발생한다.

---

## 3) 승인/기각(반려) 시나리오 (전문가 판단)

### Step 5-A. 전문가가 타스크2를 승인한다(OK)
1. 전문가가 제출된 **유저 타스크 2**를 확인한다.
2. 문제 없으면 **[승인]** 버튼을 누른다.

**결과**
- 타스크2 상태: **승인됨(완료)**
- 유저 화면: “승인 완료” 안내 표시
- (선택) 다음 단계 진행:
  - 전문가 타스크3이 자동으로 “진행 가능” 상태가 되거나
  - 전문가가 타스크3을 이어서 처리

---

### Step 5-B. 전문가가 타스크2를 기각(반려)한다(NG)
1. 전문가가 제출 내용을 확인한다.
2. 문제가 있으면 **[기각(반려)]** 버튼을 누른다.
3. **기각 사유를 입력**한다(필수 권장).
   - 예: “파일이 오래된 버전입니다. 최신 결산 자료로 다시 올려주세요.”

**결과(중요)**
- 타스크2 상태: **기각됨(재제출 필요)** 로 변경
- 유저 화면:
  - “기각됨” 표시
  - 기각 사유 표시
  - **[다시 제출하기]** 버튼 활성화

---

## 4) 재제출 루프 (기각 이후 흐름)

### Step 6. 유저가 재제출한다
1. 유저가 **[다시 제출하기]**를 눌러 수정된 자료를 제출한다.
2. 타스크2 상태는 다시 **제출됨(검토 대기)** 로 변경된다.
3. 전문가가 다시 **승인/기각** 판단을 수행한다.
4. 이 과정은 “승인될 때까지 반복”될 수 있다.

✅ 추천 정책: **기각 시 타스크2를 새로 만들지 않고, 같은 타스크2에서 재제출로 반복한다.**  
(이력이 깔끔하고 UX가 단순)

---

## 5) 화면 상태(라벨) 제안

- **요청됨**: 해야 할 일이 생성됨(전문가가 할당)
- **진행중**: 작업 중
- **제출됨(검토 대기)**: 유저가 제출했고 전문가 확인을 기다리는 중
- **승인됨(완료)**: 전문가 승인으로 완료
- **기각됨(재제출 필요)**: 반려됨 → 유저 재제출 필요

---

## 6) 추가로 정하면 좋은 정책(요구사항)

1. **기각 사유 입력 필수 여부**
   - 추천: 필수(유저가 무엇을 고쳐야 하는지 명확)
2. **재제출 횟수 제한**
   - 제한 없음 / 또는 “최대 N회” 정책 가능
3. **기한(마감) 정책**
   - 기한 초과 시 “기한 만료” 표시 여부
   - 기한 초과 시 전문가/유저 알림 여부
4. **승인/기각 권한**
   - 해당 케이스 담당 전문가(또는 같은 전문가 그룹)만 가능
