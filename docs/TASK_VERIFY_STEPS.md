# タスク機能のウェブ確認手順

> 実装したタスクAPI・TaskModal・タスク一覧表示をブラウザで確認するための順序

---

## 前提

- **開発サーバー**が起動していること（`npm run dev` → http://localhost:3000）
- **専門家（expert）またはアシスタント（assistant）**でログインできるアカウントがあること  
  （タスク作成APIは専門家のみ許可）
- 確認用の**案件（case）**が1件以上あること  
  （ない場合はヒアリングから案件作成するか、DBにテスト案件を入れておく）

---

## 確認の流れ（順番）

### Step 1. ログイン

1. ブラウザで **http://localhost:3000** を開く
2. **ログイン**（`/login`）し、**専門家 or アシスタント**のアカウントでログインする

---

### Step 2. 専門家「案件管理」へ移動

1. サイドバーなどから **「案件管理」**（専門家メイン）へ進む  
   - URL: **`/expert/management`**
2. 補助金ごとの案件数が表示された一覧が表示される

---

### Step 3. 補助金を選び、案件一覧（カンバン）を開く

1. 一覧から**任意の補助金**の行をクリック  
   - URL例: **`/expert/management/1`**（`1` は補助金ID）
2. その補助金に紐づく**案件がカンバン**（相談中・ヒアリング中・書類準備中…）で表示される

---

### Step 4. 案件カードをクリック → 案件詳細へ

1. カンバンの**いずれかの案件カード**をクリック  
   - URL例: **`/expert/management/1/123`**  
     - `1`: 補助金ID、`123`: 案件ID（caseId）
2. **案件詳細ページ**が開く（情報・タスク・書類・タイムラインのタブがある）

---

### Step 5. 「タスク」タブで一覧を確認

1. 画面上部の **「タスク」** タブをクリック
2. 次のどちらかになる：
   - **タスクが0件**  
     → 「未着手」「進行中」「完了」の3列は空、または「タスクを読み込み中...」のあと空
   - **タスクが既にある**  
     → DBから取得したタスクが3列（未着手 / 進行中 / 完了）に表示される
3. エラー時は画面上部に**赤いエラーメッセージ**が表示される（権限・案件なしなど）

---

### Step 6. 新規タスクを作成（TaskModal）

1. **「タスク追加」** ボタン（紫）をクリック
2. **新規タスク作成モーダル**が開く
3. 入力例：
   - **タスクタイプ**: 「書類アップロード」 or 「フォーム作成」
   - **タイトル**: 任意（必須）
   - **説明**: 任意
   - **期限・優先順位・担当者タイプ**（顧客/行政書士/アシスタント）など
4. **「タスク作成」** をクリック
5. 成功すると：
   - モーダルが閉じる
   - **タスク一覧が自動で再取得**され、今作ったタスクが「未着手」列などに表示される

---

### Step 7. 作成したタスクの表示確認

1. タスクタブの**未着手・進行中・完了**の3列を確認
2. 作成したタスクが **タイトル・担当者・期限・優先度** などとともに表示されていることを確認

---

## よくある確認ポイント

| 確認内容 | 想定 |
|----------|------|
| タスク一覧が空 | 初回は0件でOK。Step 6で1件作成すると表示される |
| 「タスクの取得に失敗しました」 | ログイン切れ・案件への権限なし・APIエラー。コンソールやネットワークタブで原因確認 |
| 「Only experts can create tasks」 | 顧客アカウントでログインしている。専門家/アシスタントでログインし直す |
| 案件一覧が空 | その補助金に案件がない。別補助金を選ぶか、ヒアリング等で案件を1件作成する |

---

## 確認完了の目安

- 案件詳細の「タスク」タブで、**DBのタスクが一覧表示される**
- **「タスク追加」→ 入力 → 「タスク作成」** で**新規タスクがDBに保存され、一覧に即反映される**

ここまでできていれば、タスクAPI・TaskModal・一覧表示の一通りの確認は完了です。

# 태스크 기능 웹 확인 순서 (한국어)

> 구현한 태스크 API·TaskModal·태스크 목록 표시를 브라우저에서 확인하는 순서

---

## 전제 조건

- **개발 서버**가 떠 있어야 함（`npm run dev` → http://localhost:3000）
- **전문가(expert) 또는 어시스턴트(assistant)** 계정으로 로그인 가능해야 함  
  （태스크 생성 API는 전문가만 허용）
- 확인할 **케이스(case)**가 1건 이상 있어야 함  
  （없으면 히어링으로 케이스 생성하거나, DB에 테스트 케이스를 넣어 둠）

---

## 확인 순서

### Step 1. 로그인

1. 브라우저에서 **http://localhost:3000** 접속
2. **로그인**（`/login`） 후 **전문가 또는 어시스턴트** 계정으로 로그인

---

### Step 2. 전문가「案件管理(케이스 관리)」로 이동

1. 사이드바 등에서 **「案件管理」**（전문가 메인）로 이동  
   - URL: **`/expert/management`**
2. 보조금별 케이스 수가 보이는 목록이 표시됨

---

### Step 3. 보조금 선택 후 케이스 목록(칸반) 열기

1. 목록에서 **아무 보조금** 행을 클릭  
   - URL 예: **`/expert/management/1`**（`1`은 보조금 ID）
2. 해당 보조금에 연결된 **케이스가 칸반**（상담중·히어링중·서류 준비중…）으로 표시됨

---

### Step 4. 케이스 카드 클릭 → 케이스 상세로

1. 칸반에서 **케이스 카드 하나**를 클릭  
   - URL 예: **`/expert/management/1/123`**  
     - `1`: 보조금 ID, `123`: 케이스 ID（caseId）
2. **케이스 상세 페이지**가 열림（정보·タスク·書類·타임라인 탭 있음）

---

### Step 5. 「タスク」 탭에서 목록 확인

1. 화면 상단 **「タスク」** 탭 클릭
2. 다음 중 하나:
   - **태스크 0건**  
     → 「未着手」「進行中」「完了」 3열이 비어 있거나, 「タスクを読み込み中...」 후 비어 있음
   - **이미 태스크가 있음**  
     → DB에서 가져온 태스크가 3열（未着手 / 進行中 / 完了）에 표시됨
3. 에러 시 화면 상단에 **빨간 에러 메시지** 표시（권한 없음·케이스 없음 등）

---

### Step 6. 새 태스크 생성（TaskModal）

1. **「タスク追加」** 버튼（보라색） 클릭
2. **새 태스크 작성 모달**이 열림
3. 입력 예:
   - **タスクタイプ**: 「書類アップロード」 또는 「フォーム作成」
   - **タイトル**: 임의（필수）
   - **説明**: 선택
   - **期限·優先順位·担当者タイプ**（고객/행정서사/어시스턴트） 등
4. **「タスク作成」** 클릭
5. 성공 시:
   - 모달이 닫힘
   - **태스크 목록이 자동으로 다시 불러와지고**, 방금 만든 태스크가 「未着手」 열 등에 표시됨

---

### Step 7. 만든 태스크 표시 확인

1. 태스크 탭의 **未着手·進行中·完了** 3열 확인
2. 만든 태스크가 **タイトル·担当者·期限·優先度** 등과 함께 보이는지 확인

---

## 자주 보는 확인 포인트

| 확인 내용 | 예상 |
|----------|------|
| 태스크 목록이 비어 있음 | 처음엔 0건이면 OK. Step 6에서 1건 만들면 표시됨 |
| 「タスクの取得に失敗しました」 | 로그인 만료·케이스 권한 없음·API 에러. 콘솔·네트워크 탭으로 원인 확인 |
| 「Only experts can create tasks」 | 고객 계정으로 로그인한 상태. 전문가/어시스턴트로 다시 로그인 |
| 케이스 목록이 비어 있음 | 해당 보조금에 케이스가 없음. 다른 보조금을 선택하거나, 히어링 등으로 케이스 1건 생성 |

---

## 확인 완료 기준

- 케이스 상세의 「タスク」 탭에서 **DB 태스크가 목록으로 표시됨**
- **「タスク追加」→ 입력 → 「タスク作成」** 으로 **새 태스크가 DB에 저장되고 목록에 바로 반영됨**

여기까지 되면 태스크 API·TaskModal·목록 표시까지 한 번에 확인한 것입니다.
