# 타스크(タスク) 필요 기능 리스트

> 전문가(専門家) 화면에서의 타스크 작성 기능을 위한 필요 기능 정리  
> 최종 수정: 2025-02

---

## 1. 담당 구분 (誰が担当するか)

| 구분 | 설명 | 비고 |
|------|------|------|
| **유저 담당** | 신청자(顧客・ユーザー)가 수행하는 타스크 | 예: 서류 제출, 폼 입력, 자료 업로드 |
| **전문가 담당** | 전문가(行政書士・アシスタント)가 수행하는 타스크 | 예: 서류 검토, 초안 작성, 제출 대행 |

- 타스크 생성 시 **담당 유형** 선택 필요: `user`(유저) / `expert`(전문가)
- DB: `tasks` 테이블에 `assignee_type` 또는 `responsible_role` 같은 컬럼으로 저장 검토
- 화면: 전문가가 타스크 만들 때 「顧客が対応」「専門家が対応」 등으로 선택

---

## 2. 타스크 타입 (タスクの種類)

| 타입 | 설명 | 필요 기능 |
|------|------|------------|
| **파일 업로드** | 유저/전문가가 파일을 제출하는 타스크 | • 허용 확장자(예: PDF, Word, Excel, ZIP) • 최대 용량(예: 10MB) • 다중 파일 여부 • 업로드된 파일 저장소(Supabase Storage 등) |
| **폼 작성** | 정해진 질문에 답하는 타스크 | • 질문 목록(순서, 필수/선택) • 단답/장문 등 입력 형식 • 제출 시 답변 저장(DB 테이블 또는 JSON) |
| **일반 체크** | 제출 없이 완료 체크만 하는 타스크 | • 제목・설명・기한만 있고, 완료 플래그만 갱신 |

- 현재 UI: `TaskModal`에서 이미 「書類アップロード」「フォーム作成」 두 타입 선택 가능
- 구현 시: `task_type` (예: `file` / `form` / `check`) 으로 구분

---

## 3. 공통 필드

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| **타이틀** | 필수 | 타스크명 |
| **설명** | 선택 | 상세 설명 |
| **우선순위** | 선택 | 高 / 中 / 低 (현재 `priority` 컬럼 존재) |
| **기한** | 선택 | 마감일 (현재 `deadline` 컬럼 존재) |
| **담당** | 필수 | 유저 담당 vs 전문가 담당 (위 1번) |
| **타입** | 필수 | 파일 업로드 / 폼 작성 / 일반 (위 2번) |

---

## 4. 파일 업로드 타입 시 추가 사항

- 업로드된 파일 저장 위치: **Supabase Storage** 버킷 권장
- 테이블: `task_files` 또는 `task_attachments` (task_id, file_path, uploaded_by, uploaded_at 등)
- 다운로드・삭제 권한: 담당자(유저 본인 또는 해당 케이스 전문가)만 접근 가능하도록 RLS 검토

---

## 5. 폼 작성 타입 시 추가 사항

- 질문 정의 저장: `task_form_questions` (task_id, display_order, question_text, is_required 등)
- 답변 저장: `task_form_answers` (task_id, user_id 또는 profile_id, answer_json 또는 별도 컬럼)
- 제출 전 임시 저장(드래프트) 필요 여부: 요구사항에 따라 결정

---

## 6. 화면/권한 정리

| 역할 | 타스크 생성 | 타스크 조회 | 타스크 완료/수정 |
|------|-------------|-------------|------------------|
| **전문가** | ○ (해당 케이스에 대해) | ○ | ○ (본인 담당 또는 케이스 담당 시) |
| **유저** | × | ○ (본인 케이스만) | ○ (본인 담당 타스크만) |

- 타스크 **생성**은 전문가 화면(`/expert/management/[id]/[caseId]`)에서만 가능
- 유저는 대시보드 케이스 상세에서 타스크 목록 확인 및 본인 담당 타스크 수행(파일 업로드・폼 제출・완료 체크)

---

## 7. 현재 코드베이스와의 관계

- **이미 있는 것**: `tasks` 테이블(case_id, title, description, priority, status, deadline), 유저 대시보드에서 타스크 조회・완료 토글, 전문가 측 `TaskModal` UI(타입 선택・담당・폼 질문・파일 선택 UI)
- **부족한 것**:  
  - 담당 구분(유저/전문가) DB 컬럼 및 API 반영  
  - 타스크 타입(file/form) DB 저장 및 타입별 처리  
  - 파일 업로드 시 Storage 업로드 + DB 연결  
  - 폼 질문・답변 저장 테이블 및 API  
  - TaskModal에서 「タスク作成」 클릭 시 실제 API 호출 및 DB 저장
